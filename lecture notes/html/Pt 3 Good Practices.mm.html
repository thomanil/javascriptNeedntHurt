  • Unobtrusive JavaScript
    • Separate structure, appearance and behavior
      • color code style, javascript, markup in three colors
      • Next slide, pull out in three color coded files
      • Examples
        • Show page with style, javascript, markup in one mess
        • "What's wrong w this page?"
    • Tactics for extracting javascript
      • Put javascript in separate js file
      • Put all our code inside a separate namespace
      • Use jQuery to attach event handlers etc after page loads, rather than inline onclick= etc
    • Namespacing
    • Universal access
      • JavaScript shouldn't be a barrier to entry for some users
        • Nonstandard browsers
          • Mobile browsers
          • Old browsers
          • Screenreaders
      • Progressive enhancement
        • Make sure stuff works without javascript first.
        • Gradually enrich
      • Determine if a priority at outset!
    • HOW unobtrusive?
      • Use common sense
      • One-line event handlers can be ok. More than that, separate it from your markup
  • Crossbrowser development
    • Frameworks > raw dom
    • test early and often
      • Regularly test in other browsers
        • Expect to still find bugs
        • But way fewer thanks to the practices and tools we now use!
      • Automated regression testing@
    • Clean, disciplined code
