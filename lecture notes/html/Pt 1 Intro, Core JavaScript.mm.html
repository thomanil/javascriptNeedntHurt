  • Sharp Edges

    • Global variables

      • If you don't declare with var keyword in functions, the variable ends up in global scope
      • name collisions, persistent variables

      Not knowing to avoid these is like letting your child run with scissors: it's goes well until it goes really bad

      Scope

      • Function scope only, no block scope

      Semicolon insertion

      • You can get away with forgetting ; on end of statements
      • But the browser will have to do some guesswork
      • Not all browser guess the same....

      == vs ===

      • Quiz, can you guess which of the expressions evaluate to false?
      • Nasty edge cases, hard to reason about
      • Type coercion, non-transitive
      • Just use ===
